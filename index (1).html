Compatibility

Built for
real kernels.

8.0+

Android

Tested from Oreo through the latest releases. Works on AOSP, MIUI, and most custom ROMs.

Root Frameworks

Magisk KernelSU

Boot persistence uses /data/adb/service.d/ — a directory Magisk always executes at boot, regardless of module state. When uninstalled, uninstall.sh removes the script automatically.

Classic LMK Node

Kernel requirement

/sys/module/lowmemorykiller/parameters/minfree

Your kernel must expose this path. Common on devices with kernels predating Android 10 and on many custom kernel builds.

LMKD devices not supported

Android 10+ with newer kernels use a userspace LMKD. The module detects this automatically at boot and displays a clear alert.

Quick Install

Up and running
in four steps.

No terminal needed — just Magisk and the ZIP. Verify with ADB afterward if you want to confirm the values.

01

Download the ZIP

Grab the latest release from GitHub Releases page.

02

Open Magisk or KernelSU

Navigate to Modules → Install from storage.

03

Select the ZIP and flash

The boot script is installed to /data/adb/service.d/ automatically.

04

Reboot and open the WebUI

Access the WebUI inside Magisk to select your profile. Settings survive every reboot.

Verify via ADB

adb shell
# Confirm kernel values were applied
$ cat /sys/module/lowmemorykiller/parameters/minfree
0,0,0,0,0,0
# Check which mode is saved
$ cat /data/adb/modules/lmk_controller_feerd/lmk_mode
performance
# Inspect the boot log
$ cat /data/adb/modules/lmk_controller_feerd/boot.log
--- Boot Sat Apr 5 00:00:01 -03 2026 ---
Boot completed, applying...
Mode: performance
Done: 0,0,0,0,0,0

Latest release

WebUI fully redesigned

Rebuilt interface with live sliders for all 6 minfree thresholds, real-time visual feedback, and improved touch support.

New Performance mode

A dedicated gaming profile optimized to keep critical foreground apps in RAM while aggressively clearing background processes.

Improved service.d boot persistence

Boot logic migrated from service.sh to /data/adb/service.d/ for guaranteed execution across all Magisk versions and edge cases.

Boot log with timestamp

Log now records the exact date and time, the applied mode, and all 6 confirmed minfree values from the kernel.

LMKD auto-detection

Module now detects userspace LMKD devices at boot and surfaces a clear, actionable warning instead of silently failing.

Clean uninstall

uninstall.sh now fully removes the service.d script on module removal, leaving zero residue in your system.
