Compatibility

Classic LMK vs LMKD

Android uses two different Low Memory Killer implementations. LMK Controller detects yours automatically and applies the right method.

Kernel Space

Classic LMK

The traditional kernel-level Low Memory Killer, present in older Android kernels. It reads minfree thresholds directly from a sysfs node in the kernel.

Write Method
Values are written directly to /sys/module/lowmemorykiller/parameters/minfree via a shell echo.
Presence Check
If the sysfs path exists, Classic LMK is active on your device.
Userspace Daemon

LMKD

Android's modern Low Memory Killer Daemon, running in userspace. It reads system properties to determine thresholds and is used on most devices running Android 10+.

Write Method
Properties are set via magisk resetprop --file and the daemon is restarted with a HUP signal to apply changes.
Presence Check
If ro.lmk.use_minfree_levels is blank or false, LMKD is your implementation.
🔍 How to identify your kernel type in Termux
Check for Classic LMK
ls /sys/module/lowmemorykiller/parameters/minfree
Exists → Classic LMK active
Check for LMKD
getprop ro.lmk.use_minfree_levels
Blank or false → LMKD active

Boot Flow

How it Works

LMK Controller installs a persistent boot service that runs before the Android framework, guaranteeing your LMK configuration is applied on every single reboot.

1
Module installation

Magisk or KernelSU installs the module and writes a boot service script to the system.

/data/adb/service.d/lmk_controller.sh
2
Boot execution

On every reboot, Magisk's service.d runner executes the script early in the boot process before Android frameworks initialize.

3
Kernel type detection

The script automatically checks whether Classic LMK sysfs node exists or LMKD is in use, then selects the correct write path.

4
Values applied + logged

The selected mode's minfree values are written to the kernel or daemon, and the result is recorded in the boot log for diagnostics.

/data/adb/modules/lmk_controller_feerd/boot.log
boot.log
[LMK Controller] Boot service started
→ Detecting kernel LMK type...
→ Checking Classic LMK node:
/sys/module/lowmemorykiller/parameters/minfree
→ Node not found. Using LMKD.
→ Current mode: stability
→ Applying minfree via resetprop...
ro.lmk.minfree=1024,2048,4096,8192,12288,16384
→ Sending HUP signal to lmkd...
→ lmkd restarted. Values applied.
[LMK Controller] Boot service complete ✓
Get Started

Installation

Up and running in minutes. All you need is a rooted device with Magisk or KernelSU.

1
Download the module

Grab the latest .zip from the GitHub Releases page. Always use the latest version for best compatibility.

2
Install via Magisk / KernelSU

Open your root manager (Magisk App, KernelSU App, or MMRL), go to the Modules section, and tap "Install from storage". Select the downloaded zip file.

3
Reboot your device

Reboot to allow the boot service to register in /data/adb/service.d/. The module will apply your chosen mode on this and every subsequent boot.

4
Open the WebUI in MMRL

Open MMRL and find the LMK Controller module card. Tap the WebUI button to open the control interface and select your preferred mode.

5
Apply and reboot

Select a mode in the WebUI and tap Apply. Your choice is saved and the boot service will enforce it on every reboot automatically.

Diagnostics

Verify it's working

Use Termux or any root shell to confirm LMK Controller is active and your values are applied correctly.

Termux · Verification Commands
$ # Check which kernel type you have
$ ls /sys/module/lowmemorykiller/parameters/minfree
/sys/module/lowmemorykiller/parameters/minfree  ← Classic LMK found
$ # If Classic LMK — read current values
$ cat /sys/module/lowmemorykiller/parameters/minfree
1024,2048,4096,8192,12288,16384  ← Stability mode active
$ # If LMKD — check prop values
$ getprop ro.lmk.use_minfree_levels
$ # Read the boot log for full diagnostics
$ cat /data/adb/modules/lmk_controller_feerd/boot.log
$ # Confirm boot service is registered
$ ls /data/adb/service.d/ | grep lmk
lmk_controller.sh  ← Service registered ✓
